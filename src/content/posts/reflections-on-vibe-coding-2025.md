随着 Cursor 11月13日刚刚完成一轮 $2.3B 的 [D 轮融资](https://cursor.com/blog/series-d)，投后估值达到了 $29.3B 的规模。不久之前，Lovable 年中一轮融资中，也在近期传出投后估值 $6B 的[新一轮融资](https://www.forbes.com/sites/iainmartin/2025/11/14/vibe-coding-startup-lovable-in-talks-to-raise-at-6-billion-valuation/)在接洽中。

可以毫不夸张地讲，除了百舸争流的大模型赛场， 在应用市场里，AI Coding（或 Vibe Coding）是 2025 年最大的行业趋势。而我也在用不同的方式参与到这个流程中：

- 做东西：[SeeU.Food](https://seeu.food) 一个热量记录 iOS 应用，累计收入 $87k（苹果税前）
- 找朋友：[VibeCafé](https://vibecafe.ai) 一个 Vibe 创作者组织，其中最主要的 Vibe Friends 社群做了各种活动，累计了 5k 多名成员
- 做服务：一共参与支持了 3 个 Vibe 开发者项目，从投资、推广、到业务讨论提供了不同程度的服务。其中一个 3D 小游戏引擎的项目改到 VC 模式（已退出），另外 2 个项目一个在做 🔞 漫画，一个在做设计工具，收入都在增长但规模还很小。

因此，我想要基于我这一年的参与和观察来说说我心里的 2025 年 Vibe Coding 思考。


### 思考一：绝不从开发开始，从自己喜欢、懂、热爱的事情和用户要什么开始

在 Vibe Friends 社群中，我观察和交流了非常多一模一样的故事

- 我是一个大厂的研发 Lead
- 我有个想法要 vibe 出来
- 我打开了 Cursor / Claude Code
- 我做出来了
- 我结束了

这个模式，甚至整个 VC 思维的定一个大目标然后拆解实现的思维我认为都不对。Vibe Coding 让实现一个想法的过程变得极其简单，因此从开始做为起点的创作模式是很有问题的。相反，大家最爱说的“找需求”变成了一个难点，找到正确的需求就是最重要的事情。

找需求要怎么做？我认为

- 一个人很难在自己不懂的事情上找到需求
- 需求大多数时候不是想出来的，而是身处其中，每天都在用，就能感觉到的

因此，市面上非常多的“找需求”方法论我都不太认可，相反我会建议：

1. 知道自己喜欢什么
2. 每天在自己喜欢的东西上付出几个小时专注的时间，才可能比较懂这个东西
3. 因为足够喜欢、足够懂，往往就能结交一批很懂的朋友

你有了这群朋友，大家每天的吐槽、聊天就会变的全是需求。网上有非常多的 Reddit 找需求方法论，我也很爱用 Reddit，我的建议其实就是这个逻辑：

1. 去自己喜欢的 Subreddit 里看内容
2. 不要想着营销推广什么，就基于自己懂得东西，在 Subreddit 里做一个真实的活跃用户，点赞、评论、举报、发帖，成为论坛里**真实的**活跃参与者
3. 长期观察论坛里大家反复提及的事情，需求就找到了

> 我时常开玩笑说：如果你无法用一张你想做的东西的截图在 X、TikTok、Reddit 火，大概率你做出来的产品也不会有人想用！

### 思考二：和用户在一起，不做规划，让用户想要的东西呼之欲出

传统的大厂做事模式，往往是几个人要坐下来讨论和思考非常多规划，下个季度、下个月、下一周要做什么。我认为这种模式也非常的过时和错误。

相反，你最需要做的事情，就是吸引一群喜欢你在做的东西的用户和朋友，他们越懂你，越需要你，用你的产品越多往往也就能提出越好的需求。

相比于复杂的规划，我更认可持续反复的观察大家在怎么用，以及在用户群里如何说。最好的状态就是，你能强烈的感受到大家非常需要一个东西，每次说这个事情，都能得到聊天群里大量的 +1，以及“求求了，快做吧”。理想状态下，90% 以上的需求都应该是从你的核心用户的嘴里要出来的。

但，这并不意味着创作者可以不思考，大家表达的需求和真实想要的东西往往不一样，或者容易简化。这中间转化这一层的用户、产品思考仍然是必要的。

> - 传统的壁垒：规模效应、网络效应、供应链垄断；
> 
> - Vibe的壁垒：核心用户的需求总能最快、最准确的满足并迭代出来；



### 思考三：创意的“运动员”、“自媒体”化，更及时、更长尾、更抽象

稍微抽象一点思考，Vibe Coding 让编写代码、逻辑的创作极大的提效和大众化，这就非常像是每个人都可以随时随地地拍摄和记录照片和视频。这种技术的大众化趋势，在供给侧必然是爆发的。

供给侧爆发（技术革命后），需求侧的变化也会成为必然，但需求到底是什么呢？

短视频、社交媒体、互联网，已经把大家的时间填满了，手机OS也已经成为了终端工具最好的承载平台，人类好像到了一个没啥“需求”的状态。当然不是，人性是无限的，但这一次需求的模式变化了：

> - 历史的革命：本质都是某一种需求的规模化，少数人有到人人有
> 
> - 这次的革命：可能会是某一种需求的极致个性化，人人有到我（我们）才有

而这让身处其中的 Vibe 创作者需要对我、我们每一个个体、少部分人一定要什么非常敏感，而实现就会像发短视频一样非常的快。在有一定规模受众的需求里，创作者要变的更及时、更抽象，像是运动员一样抢需求（自媒体已经到了这个阶段）；在非常小众个人的需求里，创作者要变的更“懂”，懂到只有ta才能做出来。

### 思考四：对未来（2026年）的几个认知和预判


#### 模型在把代码写对上已经不会有问题

其实到了 Claude Sonnet 4.5 开始，大多数人的业务型代码需求，只要指令准确就不太会出问题了。因此，我能观察到今年 Q3 之后，很多产品的迭代都是在如何做好类似“软件工程”，“产品流程”的事情，比如：

- Subagents 等：分工干活儿，本质就是写更多 system prompt 把事情做好
- Plan 模式：把任务规划好，问清楚需求，本质融入传统产品开发的一些必要流程
- Background Agent 等：跑场任务，本质就是 Context window 长、Context 管理得好，会压缩历史 memory，能自己干很长时间的任务还不丢失 focus


#### 生成速度会更快，更便宜

最近的 Composer-1 在速度上已经惊艳到我了，也就是说明在特定场景的模型优化是有可能极大的提效的（当然理论上必然也是对的），因此我预测 2026 年模型性价比会继续大大提升。

#### 基础大模型会把写代码自然地融入解决大众问题

目前，基础的大模型 Chatbot、搜索产品，都还是在用文字输出来解决大家的问题， 调用 code 很多时候还是需要主动要求后才做，这一步会进一步融入。

也就是说，当解决用户需求的时候，客观写一小段代码去帮你整理下数据、优化下展示形式、做一些可视化界面更好的时候，就可以非常自然的用代码来实现。

这会模糊通用大模型产品和现在所谓的“通用Agent”产品的边界，当然，我一直不太理解通用Agent产品的独特价值。甚至部分 vibe coding 产品是否能独立存在也有可能被包含进去。

#### 纯工具一定会拓展到做内容

做一个好的工具（如：Cursor）就是在做引擎，但只做引擎我判断非常难支撑目前暴涨的估值，因此 AI Coding 公司需要做一个方向判断：

- 方向一：继续深入代码逻辑能力，更底层，更专业，更强大。但它需要思考为什么自己能比几个大模型厂商做得好。我认为不太可能，因此这个方向很危险。
- 方向二：从工具发布器走向内容平台，让人人可 vibe 创作变为可能。这个是资本更爱的故事，并且大众的创作对编码复杂度的要求不会太高，也更符合他们的比较优势。


### 思考五：我们距离人人都 Vibe，还缺什么？

纯 YY，还需要个 2-3 年，但变化会加速加快：

1. 模型优化：更快、更便宜，甚至可以终端解决
2. 体裁变化：App 太重了，承载快而小的需求太慢；Web 好一些，但渲染引擎不够好；我预测是 WebGL 自动兼容 2D、3D 的一个引擎；
3. 媒介变化：一个更加围绕着群组、社群的传播媒介，是推荐Feed的阴阳两面；


`让我们一起迎接和等待 2026 年的 Vibe 吧`
